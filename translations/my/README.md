# အခြေခံ Data Science - သင်ခန်းစာ

Azure Cloud Advocates မှ Microsoft တွင် 10 ပတ်ကြာ 20 သင်ခန်းစာပါသော Data Science သင်ခန်းစာကို ပေးဆောင်ရန် ဝမ်းမြောက်ဝမ်းသာဖြစ်ပါသည်။ သင်ခန်းစာတစ်ခုစီတွင် သင်ခန်းစာမတိုင်မီနှင့် သင်ခန်းစာပြီးနောက် စမ်းမေးခွန်းများ၊ သင်ခန်းစာကို ပြီးမြောက်ရန် ရေးသားထားသော လမ်းညွှန်ချက်များ၊ ဖြေရှင်းချက်နှင့် လုပ်ငန်းတာဝန်များ ပါဝင်သည်။ ကျွန်ုပ်တို့၏ ပရောဂျက်အခြေခံ သင်ကြားမှုနည်းလမ်းသည် သင်တန်းသားများအတွက် အသစ်သော ကျွမ်းကျင်မှုများကို သင်ယူစေပြီး အတည်ပြုထားသော နည်းလမ်းဖြစ်သည်။

## Pedagogy

ဒီသင်ခန်းစာကို ဖန်တီးရာတွင် ကျွန်ုပ်တို့သည် project-based ဖြစ်စေခြင်းနှင့် မကြာခဏ စမ်းမေးခွန်းများ ပါဝင်စေခြင်းကို အခြေခံထားပါသည်။ ဒီစီးရီး၏ အဆုံးတွင် သင်တန်းသားများသည် Data Science ၏ အခြေခံအယူအဆများ၊ အကျင့်သိက္ခာဆိုင်ရာ အယူအဆများ၊ ဒေတာပြင်ဆင်မှု၊ ဒေတာနှင့် အလုပ်လုပ်နည်းများ၊ ဒေတာကို မြင်သာစေခြင်း၊ ဒေတာခွဲခြမ်းစိတ်ဖြာမှု၊ ဒေတာသုံးစွဲမှု၏ အမှန်တကယ် အသုံးချမှုများနှင့် အခြားအရာများကို သင်ယူနိုင်ပါမည်။
အပိုဆောင်းအားဖြင့်၊ အတန်းမတိုင်မီ အနိမ့်အဆင့်မေးခွန်းဖြေခြင်းသည် ကျောင်းသားများကို ခေါင်းစဉ်ကိုလေ့လာရန် ရည်ရွယ်ချက်ထားစေပြီး၊ အတန်းပြီးနောက် ဒုတိယမေးခွန်းဖြေခြင်းသည် သင်ခန်းစာကို ပိုမိုမှတ်မိစေသည်။ ဒီသင်ရိုးညွှန်းတမ်းကို အလွယ်တကူနှင့် ပျော်ရွှင်စွာ လေ့လာနိုင်ရန် ရည်ရွယ်ပြီး၊ အစအဆုံး သို့မဟုတ် အစိတ်အပိုင်းအလိုက် လေ့လာနိုင်ပါသည်။ ပရောဂျက်များသည် သေးငယ်သောအဆင့်မှ စတင်ပြီး ၁၀ ပတ်အတွင်း ပိုမိုရှုပ်ထွေးလာပါသည်။

> [Code of Conduct](CODE_OF_CONDUCT.md), [Contributing](CONTRIBUTING.md), [Translation](TRANSLATIONS.md) လမ်းညွှန်များကို ရှာဖွေပါ။ သင့်ရဲ့ အဆောက်အအုံဆန်းစစ်မှုကို ကြိုဆိုပါသည်!

## VSCode Remote - Containers
သင့်ရဲ့ ဒေသတွင်းစက်နှင့် VSCode ကို အသုံးပြု၍ VS Code Remote - Containers extension ကို အသုံးပြု၍ ဒီ repo ကို container တွင် ဖွင့်ရန် အောက်ပါအဆင့်များကို လိုက်နာပါ:

1. Development container ကို ပထမဆုံးအသုံးပြုပါက၊ သင့်စနစ်သည် [the getting started documentation](https://code.visualstudio.com/docs/devcontainers/containers#_getting-started) တွင် ဖော်ပြထားသော အလိုအလျောက်လိုအပ်ချက်များ (ဥပမာ Docker ကို ထည့်သွင်းထား) ဖြည့်ဆည်းထားသည်ကို သေချာပါစေ။

ဒီ repository ကို အသုံးပြုရန်၊ Docker volume တွင် repository ကို ဖွင့်ပါ:

**မှတ်ချက်**: အောက်ခံတွင်၊ ဒါဟာ Remote-Containers: **Clone Repository in Container Volume...** command ကို အသုံးပြု၍ Source Code ကို ဒေသတွင်း filesystem အစား Docker volume တွင် Clone လုပ်ပါမည်။ [Volumes](https://docs.docker.com/storage/volumes/) သည် container data ကို ထိန်းသိမ်းရန် အကြံပြုထားသော Mechanism ဖြစ်သည်။

သို့မဟုတ် ဒေသတွင်းတွင် Clone လုပ်ထားသော သို့မဟုတ် Download လုပ်ထားသော Repository ကို ဖွင့်ပါ:

- ဒီ repository ကို သင့်ဒေသတွင်း filesystem သို့ Clone လုပ်ပါ။
- F1 ကို နှိပ်ပြီး **Remote-Containers: Open Folder in Container...** command ကို ရွေးချယ်ပါ။
- ဒီ folder ၏ Clone လုပ်ထားသော မိတၱဴကို ရွေးချယ်ပြီး၊ container စတင်ရန် စောင့်ပါ၊ အရာများကို စမ်းသပ်ပါ။

## Offline access

ဒီ documentation ကို [Docsify](https://docsify.js.org/#/) ကို အသုံးပြု၍ offline အဖြစ် အလုပ်လုပ်စေပါ။ ဒီ repo ကို Fork လုပ်ပြီး၊ [Docsify](https://docsify.js.org/#/quickstart) ကို သင့်ဒေသတွင်းစက်တွင် ထည့်သွင်းပါ၊ ထို့နောက် ဒီ repo ၏ root folder တွင် `docsify serve` ကို ရိုက်ပါ။ Website ကို သင့် localhost `localhost:3000` တွင် port 3000 တွင် အလုပ်လုပ်စေပါမည်။

**အကြောင်းကြားချက်**:  
ဤစာရွက်စာတမ်းကို AI ဘာသာပြန်ဝန်ဆောင်မှု [Co-op Translator](https://github.com/Azure/co-op-translator) ကို အသုံးပြု၍ ဘာသာပြန်ထားပါသည်။ ကျွန်ုပ်တို့သည် တိကျမှုအတွက် ကြိုးစားနေသော်လည်း အလိုအလျောက် ဘာသာပြန်ခြင်းတွင် အမှားများ သို့မဟုတ် မတိကျမှုများ ပါဝင်နိုင်သည်ကို သတိပြုပါ။ မူရင်းဘာသာစကားဖြင့် ရေးသားထားသော စာရွက်စာတမ်းကို အာဏာတရ အရင်းအမြစ်အဖြစ် သတ်မှတ်သင့်ပါသည်။ အရေးကြီးသော အချက်အလက်များအတွက် လူက ဘာသာပြန်ခြင်းကို အကြံပြုပါသည်။ ဤဘာသာပြန်ကို အသုံးပြုခြင်းမှ ဖြစ်ပေါ်လာသော အလွဲအလွဲအမှားများ သို့မဟုတ် အနားယူမှုများအတွက် ကျွန်ုပ်တို့သည် တာဝန်မယူပါ။
